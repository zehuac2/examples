_blank opens a new browsing context — usually a new tab or window depending on browser and user settings. Each call with _blank creates a new context unless you reuse a named target. For security, add rel="noopener noreferrer" to links or set openedWindow.opener = null so the new page cannot access window.opener.

_parent loads the URL into the immediate parent of the current browsing context (the containing frame). If there is no parent (top-level), _parent behaves like _self and replaces the current document.

_self loads the URL in the current browsing context, replacing this document — equivalent to navigating normally in the same tab or frame.

_top targets the topmost browsing context, breaking out of any nested frames and loading the URL into the top-level window. If the current document is already top-level, _top behaves like _self.

Named target: "myWindow" opens or reuses a window/tab with the exact name myWindow. Repeated calls with this same name will reuse and focus that same context instead of creating new ones — useful for popups you want to update rather than multiply.

Notes

Notes for this example: calling window.open('', target, 'width=400,height=300') opens an about:blank document in the chosen context and then uses document.write to populate it. This works when the opened context is same-origin or when you're writing immediately to the newly-created blank document. You cannot script or inspect cross-origin pages due to the same-origin policy. Browser popup blockers and user preferences can also change whether a new window or tab is created.

Best practice: prefer named targets for reusable windows, and for _blank use rel="noopener noreferrer" or explicitly null out opener when using window.open to avoid exposing the opener page to the new page.
